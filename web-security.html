üîü OWASP Top 10 Vulnerabilities (2021)

The most critical web application security risks according to the Open Web Application Security Project:

1 Broken Access Control

Users can access unauthorized functionality or data. Accounts can be compromised, permissions bypassed.

Examples: Direct URL access, privilege escalation, metadata manipulation

2 Cryptographic Failures

Sensitive data transmitted or stored without proper encryption protection.

Examples: Plain text passwords, weak encryption, HTTP instead of HTTPS

3 Injection Attacks

Untrusted data sent to interpreters as part of commands or queries.

Examples: SQL injection, NoSQL injection, OS command injection, LDAP injection

4 Insecure Design

Missing or ineffective control design flaws and architectural weaknesses.

Examples: Missing security controls, insecure design patterns

5 Security Misconfiguration

Insecure default configurations, incomplete setups, open cloud storage.

Examples: Default accounts, unnecessary features enabled, verbose error messages

6 Vulnerable Components

Using components with known vulnerabilities or outdated versions.

Examples: Outdated libraries, unpatched frameworks, unused dependencies

7 Authentication Failures

Application functions related to authentication and session management are implemented incorrectly.

Examples: Weak passwords, session fixation, missing MFA

8 Software Integrity Failures

Code and infrastructure that do not protect against integrity violations.

Examples: Unsigned software updates, insecure CI/CD pipelines

9 Logging Failures

Insufficient logging and monitoring, coupled with missing or ineffective integration.

Examples: Missing security logs, inadequate log protection, slow incident response

10 Server-Side Request Forgery (SSRF)

Web application fetches remote resources without validating user-supplied URLs.

Examples: Internal service access, cloud metadata access, port scanning

üèÜ Security Best Practices

üõ°Ô∏è Input Validation

  • Validate all user input server-side
  • Use allowlists over blocklists
  • Sanitize and escape output
  • Use parameterized queries
  • Implement Content Security Policy (CSP)

üîê Authentication

  • Enforce strong password policies
  • Implement multi-factor authentication
  • Use secure session management
  • Lock accounts after failed attempts
  • Never store passwords in plain text

üîÑ Access Control

  • Implement principle of least privilege
  • Use role-based access control (RBAC)
  • Verify permissions on every request
  • Never rely on client-side authorization
  • Log all access attempts

üì° Communication

  • Always use HTTPS in production
  • Implement HTTP Strict Transport Security
  • Validate SSL certificates
  • Use secure cookies (Secure, HttpOnly flags)
  • Implement proper CORS policies

üîß Configuration

  • Remove default accounts and passwords
  • Disable unnecessary features/services
  • Keep all software updated
  • Use security headers
  • Regular security audits

üìä Monitoring

  • Log security-relevant events
  • Monitor for suspicious patterns
  • Set up real-time alerts
  • Regular security testing
  • Incident response plan
