Initial Setup

# Configure Git (first time setup)
git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"
git config --global init.defaultBranch main

# Initialize a repository
git init
git init --bare  # Create bare repository (for servers)

# Clone a repository
git clone https://github.com/user/repo.git
git clone https://github.com/user/repo.git my-folder
git clone --depth 1 https://github.com/user/repo.git  # Shallow clone

# Check status and configuration
git status
git config --list
git config user.name

Basic Operations

# Adding files
git add file.txt                    # Add specific file
git add .                          # Add all files in current directory
git add -A                         # Add all files (including deleted)
git add -u                         # Add only modified files
git add -p                         # Interactive staging (patch mode)

# Committing changes
git commit -m "Commit message"     # Commit with message
git commit -am "Message"           # Add and commit modified files
git commit --amend                 # Amend last commit
git commit --amend -m "New message" # Amend with new message
git commit --no-edit --amend       # Amend without changing message

# Viewing changes
git diff                           # Changes in working directory
git diff --staged                  # Changes in staging area
git diff HEAD~1                    # Compare with previous commit
git diff branch1..branch2          # Compare branches
git diff --name-only               # Show only file names

Branching

Git Branching Model

gitgraph commit id: "Initial" branch develop commit id: "Feature prep" branch feature commit id: "Feature work" commit id: "Feature complete" checkout develop merge feature commit id: "Integration" checkout main merge develop commit id: "Release"

Branch Operations

# Creating branches
git branch feature-branch          # Create new branch
git checkout -b feature-branch     # Create and switch to branch
git switch -c feature-branch       # Modern way to create and switch

# Switching branches
git checkout main                  # Switch to main branch
git switch main                    # Modern way to switch
git checkout -                     # Switch to previous branch

# Listing branches
git branch                         # List local branches
git branch -r                      # List remote branches
git branch -a                      # List all branches
git branch -v                      # List with last commit info
git branch --merged               # List merged branches
git branch --no-merged            # List unmerged branches

# Deleting branches
git branch -d feature-branch       # Delete merged branch
git branch -D feature-branch       # Force delete branch
git push origin --delete feature-branch  # Delete remote branch

# Renaming branches
git branch -m old-name new-name    # Rename branch
git branch -m new-name             # Rename current branch

# Branch tracking
git branch -u origin/main          # Set upstream branch
git branch --set-upstream-to=origin/main main

🌟 Feature Branches

Create isolated environments for developing new features without affecting main codebase.

git checkout -b feature/user-authentication

🐛 Hotfix Branches

Quickly fix critical bugs in production by branching from main.

git checkout -b hotfix/security-patch

🚀 Release Branches

Prepare releases with final testing and bug fixes before merging to main.

git checkout -b release/v1.2.0

Merging & Rebasing

Merging Strategies

# Basic merge
git merge feature-branch           # Merge feature branch into current
git merge --no-ff feature-branch   # Force merge commit (no fast-forward)
git merge --squash feature-branch  # Squash all commits into one

# Merge with conflict resolution
git merge feature-branch
# If conflicts occur:
git status                         # See conflicted files
# Edit files to resolve conflicts
git add resolved-file.txt
git commit                         # Complete the merge

# Abort merge
git merge --abort                  # Cancel ongoing merge

# Fast-forward vs No-fast-forward
git merge --ff-only feature        # Only if fast-forward possible
git merge --no-ff feature          # Always create merge commit

Rebasing

# Basic rebase
git rebase main                    # Rebase current branch onto main
git rebase main feature-branch     # Rebase feature onto main

# Interactive rebase
git rebase -i HEAD~3               # Interactive rebase last 3 commits
git rebase -i main                 # Interactive rebase onto main

# Interactive rebase actions:
# pick    = use commit as-is
# reword  = use commit but edit message
# edit    = use commit but stop for amending
# squash  = meld into previous commit
# fixup   = like squash but discard commit message
# drop    = remove commit

# Continue/abort rebase
git rebase --continue              # Continue after resolving conflicts
git rebase --abort                 # Abort rebase operation
git rebase --skip                  # Skip current commit

# Rebase onto different branch
git rebase --onto main develop feature  # Rebase feature commits that are not in develop

Remote Repositories

Remote Management

# Adding remotes
git remote add origin https://github.com/user/repo.git
git remote add upstream https://github.com/original/repo.git

# Listing remotes
git remote                         # List remote names
git remote -v                      # List with URLs
git remote show origin             # Show remote details

# Changing remotes
git remote set-url origin https://github.com/user/new-repo.git
git remote rename origin upstream  # Rename remote
git remote remove upstream         # Remove remote

# Fetching and pulling
git fetch                          # Fetch all remotes
git fetch origin                   # Fetch specific remote
git fetch origin main             # Fetch specific branch

git pull                          # Fetch and merge current branch
git pull origin main              # Pull specific branch
git pull --rebase                 # Pull with rebase instead of merge
git pull --ff-only                # Only pull if fast-forward possible

# Pushing
git push                          # Push current branch
git push origin main              # Push to specific remote/branch
git push -u origin feature        # Push and set upstream
git push --all                    # Push all branches
git push --tags                   # Push all tags
git push --force                  # Force push (dangerous!)
git push --force-with-lease       # Safer force push

Tracking Branches

# Set up tracking
git branch --set-upstream-to=origin/main main
git checkout -b feature origin/feature  # Create tracking branch

# Check tracking information
git branch -vv                     # Show tracking info for all branches
git status                         # Shows ahead/behind info

# Prune remote branches
git remote prune origin            # Remove stale remote-tracking branches
git fetch -p                       # Fetch with prune

History & Changes

Viewing History

# Basic log
git log                           # Full commit history
git log --oneline                 # Condensed one-line format
git log --graph                   # ASCII graph of branches
git log --all --graph --oneline   # Graph of all branches

# Filtering logs
git log -n 10                     # Last 10 commits
git log --since="2023-01-01"      # Commits since date
git log --until="1 week ago"      # Commits until timeframe
git log --author="John Doe"       # Commits by author
git log --grep="bug fix"          # Commits with message containing text

# File-specific history
git log filename.txt              # History of specific file
git log -p filename.txt           # History with patches
git log --follow filename.txt     # Follow file renames

# Advanced log formatting
git log --pretty=format:"%h %an %ar %s"  # Custom format
git log --stat                    # Show changed files statistics
git log --shortstat               # Show insertion/deletion summary
git log --name-only               # Show only changed file names
git log --name-status             # Show changed files with status

Examining Changes

# Show specific commit
git show HEAD                     # Show last commit
git show commit-hash              # Show specific commit
git show HEAD~2                   # Show commit 2 steps back
git show HEAD:filename.txt        # Show file from specific commit

# Compare commits
git diff HEAD~1..HEAD             # Compare last two commits
git diff commit1..commit2         # Compare two commits
git diff main..feature            # Compare branches

# Find when something changed
git log -S "function_name"        # Find commits that changed content
git log -G "regex_pattern"        # Find commits matching regex
git blame filename.txt            # Show who changed each line
git annotate filename.txt         # Alternative to blame

# Search in history
git log --all --full-history -- filename.txt  # Full history including deletions
git rev-list --all | xargs git grep "search_term"  # Search in all commits

References and Tags

# Working with tags
git tag                           # List all tags
git tag v1.0.0                    # Create lightweight tag
git tag -a v1.0.0 -m "Version 1.0.0"  # Create annotated tag
git tag -d v1.0.0                 # Delete local tag
git push origin v1.0.0            # Push tag to remote
git push origin --tags            # Push all tags
git push origin :refs/tags/v1.0.0 # Delete remote tag

# References
git show-ref                      # Show all references
git symbolic-ref HEAD             # Show current branch reference
git rev-parse HEAD                # Get commit hash of HEAD
git rev-parse --short HEAD        # Get short commit hash

Undoing Changes

Reset Commands

# Reset types
git reset --soft HEAD~1           # Undo commit, keep changes staged
git reset --mixed HEAD~1          # Undo commit and staging (default)
git reset --hard HEAD~1           # Undo everything (dangerous!)

# Reset specific files
git reset filename.txt            # Unstage file
git reset HEAD filename.txt       # Unstage file (explicit)
git checkout -- filename.txt      # Discard working directory changes
git restore filename.txt          # Modern way to discard changes
git restore --staged filename.txt # Modern way to unstage

# Reset to specific commit
git reset --hard commit-hash      # Reset to specific commit
git reset --hard origin/main      # Reset to match remote

# Reflog (recovery)
git reflog                        # Show reference log
git reset --hard HEAD@{2}         # Reset using reflog reference

Revert and Clean

# Revert commits (safe for shared history)
git revert HEAD                   # Create commit that undoes last commit
git revert HEAD~2                 # Revert specific commit
git revert --no-edit HEAD         # Revert without opening editor
git revert HEAD~3..HEAD           # Revert range of commits

# Clean untracked files
git clean -n                      # Dry run - show what would be deleted
git clean -f                      # Force delete untracked files
git clean -fd                     # Delete untracked files and directories
git clean -fX                     # Delete only ignored files
git clean -fx                     # Delete untracked and ignored files

# Stashing changes
git stash                         # Stash current changes
git stash push -m "Work in progress"  # Stash with message
git stash list                    # List all stashes
git stash show                    # Show stash contents
git stash pop                     # Apply and remove last stash
git stash apply                   # Apply stash without removing
git stash drop                    # Delete stash
git stash clear                   # Delete all stashes

# Stash specific files
git stash push -m "message" -- file1.txt file2.txt
git stash push --keep-index       # Stash everything except staged files

🔄 Use Reset Carefully

Understand the difference between --soft, --mixed, and --hard. Never use --hard on shared branches.

↩️ Prefer Revert for Shared History

Use git revert instead of reset for commits that have been pushed to shared repositories.

📦 Leverage Stash for Context Switching

Use stash to temporarily save work when switching branches or pulling updates.

Submodules

What are Git Submodules?

Submodules allow you to include one Git repository as a subdirectory of another Git repository. This enables you to keep a Git repository as a subtree of another Git repository while maintaining separate history for both projects.

Submodule Operations

# Adding submodules
git submodule add https://github.com/user/library.git lib/library
git submodule add -b main https://github.com/user/theme.git themes/custom

# Initialize and update submodules
git submodule init                 # Initialize submodule configuration
git submodule update               # Update submodules to recorded commits
git submodule update --init        # Initialize and update in one command
git submodule update --init --recursive  # Include nested submodules

# Clone repository with submodules
git clone --recursive https://github.com/user/project.git
git clone --recurse-submodules https://github.com/user/project.git

# Update submodules to latest commits
git submodule update --remote     # Update to latest remote commits
git submodule update --remote --merge  # Merge latest changes
git submodule update --remote --rebase # Rebase local changes

# Working with submodules
git submodule status               # Show submodule status
git submodule summary             # Show summary of changes
git submodule foreach 'git status' # Run command in each submodule
git submodule foreach 'git pull origin main'  # Pull latest in all submodules

# Removing submodules
git submodule deinit lib/library   # Deinitialize submodule
git rm lib/library                 # Remove submodule from index
rm -rf .git/modules/lib/library    # Remove submodule git directory

Submodule Configuration

# .gitmodules file example
[submodule "lib/library"]
    path = lib/library
    url = https://github.com/user/library.git
    branch = main

[submodule "themes/custom"]
    path = themes/custom
    url = https://github.com/user/theme.git
    branch = develop

# Configure submodule behavior
git config submodule.lib/library.update merge
git config submodule.lib/library.branch main

# Global submodule settings
git config --global submodule.recurse true  # Auto-recurse submodules
git config --global diff.submodule log      # Better submodule diffs

📚 External Libraries

Include external libraries or dependencies as submodules to maintain version control.

Vendor libraries, shared utilities

🎨 Shared Resources

Share themes, configurations, or assets across multiple projects.

UI themes, configuration files

🏗️ Modular Architecture

Organize large projects into smaller, manageable components.

Microservices, plugin systems

⚠️ Submodule Gotchas

  • Submodules are pinned to specific commits, not branches
  • Changes inside submodules need to be committed separately
  • Team members must run git submodule update after pulling
  • Removing submodules requires multiple steps
  • Nested submodules can be complex to manage

Advanced Commands

Cherry-pick and Bisect

# Cherry-pick commits
git cherry-pick commit-hash       # Apply specific commit to current branch
git cherry-pick commit1..commit3  # Cherry-pick range of commits
git cherry-pick --no-commit hash  # Apply changes without committing
git cherry-pick --continue        # Continue after resolving conflicts
git cherry-pick --abort           # Abort cherry-pick operation

# Binary search for bugs
git bisect start                  # Start bisection
git bisect bad                    # Mark current commit as bad
git bisect good v1.0              # Mark v1.0 as good
git bisect reset                  # End bisection session

# Automated bisect
git bisect run test.sh            # Run script to test each commit

Working with Patches

# Create patches
git format-patch -1              # Create patch for last commit
git format-patch HEAD~3          # Create patches for last 3 commits
git format-patch main..feature   # Create patches for branch difference

# Apply patches
git apply patch-file.patch       # Apply patch without committing
git am patch-file.patch          # Apply patch with commit (from format-patch)
git am --3way patch-file.patch   # Apply with 3-way merge if conflicts

# Diff and patch
git diff > changes.patch         # Create diff patch
git apply changes.patch          # Apply diff patch

Git Hooks and Automation

# Common Git hooks (in .git/hooks/)
# pre-commit - Run before commit
# post-commit - Run after commit
# pre-push - Run before push
# post-receive - Run on server after receive

# Example pre-commit hook (prevent large files)
#!/bin/sh
# Check for files larger than 5MB
find . -size +5M -type f -exec ls -lh {} \; | awk '{ print $9 ": " $5 }'
if [ $? -eq 0 ]; then
    echo "Large files found. Commit rejected."
    exit 1
fi

# Worktrees (multiple working directories)
git worktree add ../feature-branch feature-branch
git worktree list                # List all worktrees
git worktree remove ../feature-branch  # Remove worktree

# Sparse checkout (partial repository)
git config core.sparseCheckout true
echo "src/*" > .git/info/sparse-checkout
git read-tree -m -u HEAD

git flow init

# Feature branches
git flow feature start new-feature    # Start new feature

Best Practices

📝 Write Good Commit Messages

Use imperative mood, keep first line under 50 characters, explain "why" not "what".

Fix user authentication timeout issue
Add user profile caching to improve performance

🌿 Branch Naming Conventions

Use descriptive, consistent branch names with categories.

feature/user-authentication
bugfix/login-validation
hotfix/security-patch

🔄 Small, Focused Commits

Make atomic commits that represent single logical changes.

🚫 Never Rewrite Shared History

Avoid force pushing to shared branches. Use revert instead of reset for shared commits.

🧪 Test Before Committing

Run tests and ensure code quality before committing changes.

📋 Use .gitignore Effectively

Properly configure .gitignore to exclude build artifacts, dependencies, and sensitive files.

Git Configuration Best Practices

# Recommended global configuration
git config --global core.autocrlf input    # Handle line endings
git config --global core.editor vim        # Set preferred editor
git config --global merge.tool vimdiff     # Set merge tool
git config --global push.default simple    # Push only current branch
git config --global pull.rebase true       # Rebase on pull by default
git config --global rebase.autoStash true  # Auto-stash during rebase

# Useful aliases
git config --global alias.co checkout
git config --global alias.br branch
git config --global alias.ci commit
git config --global alias.st status
git config --global alias.unstage 'reset HEAD --'
git config --global alias.last 'log -1 HEAD'
git config --global alias.visual '!gitk'
git config --global alias.lg "log --graph --pretty=format:'%Cred%h%Creset -%C(yellow)%d%Creset %s %Cgreen(%cr) %C(bold blue)<%an>%Creset' --abbrev-commit"

# Security settings
git config --global user.signingkey YOUR_GPG_KEY_ID
git config --global commit.gpgsign true    # Sign commits with GPG

⚠️ Common Git Mistakes to Avoid

  • Committing sensitive information (passwords, API keys)
  • Making commits too large or unfocused
  • Not pulling before pushing to shared branches
  • Using git push --force on shared branches
  • Not backing up important work before complex operations
  • Ignoring merge conflicts or resolving them incorrectly
  • Not using meaningful branch names or commit messages

Emergency Git Commands

# Oh no! I committed to wrong branch
git reset HEAD~1                   # Undo commit (keep changes)
git stash                          # Stash changes
git checkout correct-branch        # Switch to correct branch
git stash pop                      # Apply changes
git add . && git commit           # Commit to correct branch

# Oh no! I need to fix the last commit message
git commit --amend                 # Edit last commit message

# Oh no! I accidentally deleted a file
git checkout HEAD -- filename.txt # Restore file from last commit

# Oh no! I need to undo a merge
git reset --merge                  # Undo merge (if not pushed)
git revert -m 1 merge-commit-hash  # Revert merge (if pushed)

# Oh no! I need to find a lost commit
git reflog                         # Find lost commits
git cherry-pick lost-commit-hash   # Recover lost commit
