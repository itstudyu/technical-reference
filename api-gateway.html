🚪 API Gateway

The central hub that manages, secures, and orchestrates all your API traffic. Think of it as the front door and traffic controller for your microservices architecture.

🤔 What is an API Gateway?

An API Gateway is like a smart receptionist for your backend services. It sits between your clients (mobile apps, websites, partners) and your backend services, handling all the complexity of routing, security, monitoring, and more.

🔍 Detailed Request Flow

1

Client Sends Request

Mobile app or web client sends HTTP request to API Gateway endpoint

POST
https://api.example.com/v1/orders
Authorization: Bearer eyJ0eXAiOiJKV1Q...
Content-Type: application/json

{"product_id": 123, "quantity": 2}
2

SSL/TLS Termination

Gateway handles HTTPS encryption/decryption, validates certificates

✅ Security: Centralizes certificate management and encryption handling
3

Authentication & Authorization

Validates API keys, JWT tokens, or OAuth credentials

// JWT Token Validation
{
  "sub": "user123",
  "scope": ["orders:read", "orders:write"],
  "exp": 1640995200
}

✅ Valid - User authorized for orders operations
4

Rate Limiting Check

Checks if client has exceeded request limits (e.g., 1000 requests/hour)

⚠️ Rate Limit: 847/1000 requests used this hour - Request allowed
5

Request Transformation

Modifies request format, adds headers, validates input schema

// Added by Gateway:
X-Request-ID: req_abc123
X-Client-IP: 192.168.1.100
X-User-ID: user123

// Transformed payload:
{
  "product_id": 123,
  "quantity": 2,
  "user_id": "user123",
  "timestamp": "2023-12-01T10:30:00Z"
}
6

Service Discovery & Routing

Determines which backend service should handle the request and routes accordingly

Route: /v1/orders → Order Service
Backend: http://order-service:3000
Load Balance: Round-robin (server 2 of 3)
Health Check: ✅ Healthy
7

Backend Processing

Backend service processes the request and returns response

🔄 Processing: Order service validates product, checks inventory, creates order
8

Response Processing

Gateway processes backend response, adds headers, handles errors

HTTP/1.1 201 Created
Content-Type: application/json
X-Request-ID: req_abc123
X-Response-Time: 145ms
Cache-Control: no-cache

{
  "order_id": "ord_xyz789",
  "status": "created",
  "total": 29.99
}
9

Logging & Monitoring

Records request/response details, metrics, and performance data

📊 Logged: Request ID, Duration: 145ms, Status: 201, Client IP, User ID
10

Return to Client

Final response sent back to the client with all transformations applied

✅ Success: Order created successfully, client receives confirmation

⚖️ With vs Without API Gateway

❌ Without API Gateway

  • Clients connect directly to each service
  • Each service handles its own security
  • Scattered authentication logic
  • Complex client-side routing
  • Inconsistent error handling
  • No centralized monitoring
  • Difficult API versioning
  • CORS issues with multiple domains
Problems:
Tight coupling, security gaps, monitoring chaos, client complexity

✅ With API Gateway

  • Single entry point for all APIs
  • Centralized security & authentication
  • Consistent authentication across services
  • Simple client implementation
  • Standardized error responses
  • Comprehensive monitoring & analytics
  • Easy API versioning & deprecation
  • CORS handled in one place
Benefits:
Loose coupling, unified security, centralized monitoring, simple clients

💻 Simple Implementation Example

Node.js + Express API Gateway

const express = require('express'); const httpProxy = require('http-proxy-middleware'); const rateLimit = require('express-rate-limit'); const jwt = require('jsonwebtoken'); const app = express(); // Rate limiting middleware const limiter = rateLimit({ windowMs: 15 * 60 * 1000, // 15 minutes max: 100, // limit each IP to 100 requests per windowMs message: { error: 'Too many requests, please try again later' } }); // Authentication middleware const authenticateToken = (req, res, next) => { const authHeader = req.headers['authorization']; const token = authHeader && authHeader.split(' ')[1]; if (!token) { return res.status(401).json({ error: 'Access token required' }); } jwt.verify(token, process.env.JWT_SECRET, (err, user) => { if (err) return res.status(403).json({ error: 'Invalid token' }); req.user = user; next(); }); }; // Logging middleware const requestLogger = (req, res, next) => { const start = Date.now(); res.on('finish', () => { const duration = Date.now() - start; console.log({ method: req.method, url: req.url, status: res.statusCode, duration: `${duration}ms`, userAgent: req.get('User-Agent'), ip: req.ip }); }); next(); }; // Apply global middleware app.use(express.json()); app.use(limiter); app.use(requestLogger); // Health check endpoint app.get('/health', (req, res) => { res.json({ status: 'healthy', timestamp: new Date().toISOString() }); }); // User service proxy app.use('/api/users', authenticateToken, httpProxy({ target: 'http://user-service:3001', changeOrigin: true, pathRewrite: { '^/api/users': '' }, onProxyReq: (proxyReq, req, res) => { // Add user context to request proxyReq.setHeader('X-User-ID', req.user.id); proxyReq.setHeader('X-User-Role', req.user.role); } }) ); // Order service proxy app.use('/api/orders', authenticateToken, httpProxy({ target: 'http://order-service:3002', changeOrigin: true, pathRewrite: { '^/api/orders': '' }, onError: (err, req, res) => { console.error('Order service error:', err); res.status(503).json({ error: 'Order service unavailable', requestId: req.headers['x-request-id'] }); } }) ); // Payment service proxy with circuit breaker pattern let paymentServiceDown = false; let failureCount = 0; const MAX_FAILURES = 5; app.use('/api/payments', authenticateToken, (req, res, next) => { if (paymentServiceDown) { return res.status(503).json({ error: 'Payment service temporarily unavailable' }); } next(); }, httpProxy({ target: 'http://payment-service:3003', changeOrigin: true, pathRewrite: { '^/api/payments': '' }, onError: (err, req, res) => { failureCount++; if (failureCount >= MAX_FAILURES) { paymentServiceDown = true; // Reset after 30 seconds setTimeout(() => { paymentServiceDown = false; failureCount = 0; }, 30000); } res.status(503).json({ error: 'Payment processing failed' }); } }) ); // Global error handler app.use((err, req, res, next) => { console.error('Gateway error:', err); res.status(500).json({ error: 'Internal gateway error', requestId: req.headers['x-request-id'] }); }); // 404 handler app.use('*', (req, res) => { res.status(404).json({ error: 'Endpoint not found', availableEndpoints: ['/api/users', '/api/orders', '/api/payments'] }); }); const PORT = process.env.PORT || 3000; app.listen(PORT, () => { console.log(`🚪 API Gateway running on port ${PORT}`); });

🎯 Best Practices

🔒

Security First

  • Always validate and sanitize inputs
  • Implement proper authentication
  • Use HTTPS everywhere
  • Apply rate limiting
  • Log security events
⚡

Performance

  • Implement intelligent caching
  • Use connection pooling
  • Enable response compression
  • Monitor response times
  • Implement circuit breakers
📊

Monitoring

  • Log all requests and responses
  • Track performance metrics
  • Set up health checks
  • Monitor error rates
  • Alert on anomalies
🔄

Reliability

  • Implement retry mechanisms
  • Use timeouts appropriately
  • Plan for graceful degradation
  • Test failure scenarios
  • Have rollback strategies
