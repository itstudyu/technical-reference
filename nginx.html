ü§î What is NGINX?

NGINX (pronounced "engine-x") is a high-performance web server that can also work as a reverse proxy, load balancer, and HTTP cache. It's used by Netflix, GitHub, WordPress.com, and millions of other websites.

üìã Step-by-Step Process

1

Client Sends HTTP Request

User's browser sends GET /api/users to example.com

2

NGINX Receives Request

NGINX listens on port 80/443 and receives the HTTP request

3

NGINX Processes Request

Checks configuration, applies rules, adds/modifies headers

4

NGINX Forwards to Backend

Sends request to backend server (e.g., http://localhost:3000)

5

Backend Processes & Responds

Backend server processes request and sends HTTP response back to NGINX

6

NGINX Processes Response

May cache response, add security headers, or compress content

7

NGINX Returns to Client

Sends final HTTP response back to the user's browser

‚öôÔ∏è Basic Configuration

Simple Website Server

Basic configuration to serve a static website:

server {
    listen 80;
    server_name example.com www.example.com;
    root /var/www/html;
    index index.html index.htm;

    location / {
        try_files $uri $uri/ =404;
    }
}

Reverse Proxy Configuration

Forward requests to a backend application:

server {
    listen 80;
    server_name api.example.com;

    location / {
        proxy_pass http://localhost:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}

Load Balancer Configuration

Distribute requests across multiple servers:

upstream backend {
    server 192.168.1.10:80;
    server 192.168.1.11:80;
    server 192.168.1.12:80;
}

server {
    listen 80;
    server_name example.com;

    location / {
        proxy_pass http://backend;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}

üìã HTTP to HTTPS Conversion Process

1

Client Sends HTTP Request

User types http://example.com or clicks HTTP link

GET / HTTP/1.1
Host: example.com
User-Agent: Mozilla/5.0...
2

NGINX Receives HTTP Request (Port 80)

NGINX server block configured to listen on port 80 receives the request

server {
    listen 80;
    server_name example.com;
    # This block handles HTTP requests
}
3

NGINX Issues 301 Redirect

NGINX automatically redirects HTTP to HTTPS using a 301 permanent redirect

server {
    listen 80;
    server_name example.com;
    return 301 https://$server_name$request_uri;
}
4

Client Receives Redirect Response

Browser receives 301 redirect and understands it needs to use HTTPS

HTTP/1.1 301 Moved Permanently
Location: https://example.com/
Server: nginx/1.18.0
5

Client Initiates HTTPS Connection

Browser automatically makes new request to HTTPS (port 443)

TLS Handshake Initiated:
1. ClientHello ‚Üí Server
2. ServerHello ‚Üê Server
3. Certificate ‚Üê Server
4. Key Exchange ‚Üí ‚Üê Both
5. üîí Encrypted Connection Established
6

NGINX Handles HTTPS Request (Port 443)

NGINX SSL server block receives and decrypts the HTTPS request

server {
    listen 443 ssl http2;
    server_name example.com;
    ssl_certificate /path/to/cert.crt;
    ssl_certificate_key /path/to/key.key;
}
7

SSL Termination & Backend Communication

NGINX terminates SSL and communicates with backend via HTTP (internal network)

Why HTTP internally?
‚Ä¢ Backend servers are on secure internal network
‚Ä¢ Reduces CPU overhead on application servers
‚Ä¢ NGINX handles all SSL/TLS complexity
‚Ä¢ Easier backend development and debugging
8

Encrypted Response Back to Client

NGINX encrypts response and sends it back via HTTPS

HTTPS/1.1 200 OK
Strict-Transport-Security: max-age=31536000
Content-Type: text/html

HTTPS with SSL Certificate

server {
    listen 443 ssl http2;
    server_name example.com;

    ssl_certificate /path/to/certificate.crt;
    ssl_certificate_key /path/to/private.key;

    # Security settings
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;
    ssl_prefer_server_ciphers on;

    location / {
        root /var/www/html;
        index index.html;
    }
}

# Redirect HTTP to HTTPS
server {
    listen 80;
    server_name example.com;
    return 301 https://$server_name$request_uri;
}

üöÄ Performance Optimization

Gzip Compression

Compress files to reduce bandwidth usage:

http {
    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml text/javascript;
    gzip_min_length 1000;
}

Caching

Cache static files for better performance:

location ~* \.(jpg|jpeg|png|gif|ico|css|js)$ {
    expires 1y;
    add_header Cache-Control "public, immutable";
}

location / {
    add_header Cache-Control "no-cache";
}

Rate Limiting

Protect against abuse and DDoS:

http {
    limit_req_zone $binary_remote_addr zone=api:10m rate=10r/s;
}

server {
    location /api/ {
        limit_req zone=api burst=20 nodelay;
        proxy_pass http://backend;
    }
}

üîß Real-World Configuration Examples

React App with API Backend

server {
    listen 80;
    server_name myapp.com;

    # Serve React app
    location / {
        root /var/www/react-app/build;
        try_files $uri $uri/ /index.html;
    }

    # Proxy API requests
    location /api/ {
        proxy_pass http://localhost:3001;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}

WordPress with PHP-FPM

server {
    listen 80;
    server_name wordpress.com;
    root /var/www/wordpress;
    index index.php index.html;

    location / {
        try_files $uri $uri/ /index.php?$args;
    }

    location ~ \.php$ {
        fastcgi_pass unix:/var/run/php/php7.4-fpm.sock;
        fastcgi_index index.php;
        include fastcgi_params;
        fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
    }
}
