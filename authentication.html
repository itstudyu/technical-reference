Overview

Authentication is the process of verifying the identity of users or systems. In web applications, there are several methods to implement authentication, each with its own advantages and use cases.

Key Authentication Methods

  • Session-Based Authentication: Traditional server-side session management
  • JWT (JSON Web Tokens): Stateless token-based authentication
  • OAuth 2.0: Authorization framework for third-party access

Authentication Flow Overview

Session-Based Authentication

Traditional authentication method where user credentials are verified and a session is created on the server. The session ID is stored in a cookie and sent with each request.

Session Authentication Flow

sequenceDiagram participant C as Client participant S as Server participant DB as Session Store C->>S: POST /login (username, password) S->>S: Validate credentials S->>DB: Create session DB-->>S: Session ID S->>C: Set cookie with session ID Note over C,S: Subsequent requests C->>S: Request with session cookie S->>DB: Validate session ID DB-->>S: Session data S->>C: Authenticated response

‚úÖ Advantages

  • Server-side control over sessions
  • Easy to implement and understand
  • Session data stored securely on server
  • Can easily revoke sessions

‚ùå Disadvantages

  • Not suitable for stateless applications
  • Memory usage grows with users
  • Challenging for distributed systems
  • CSRF vulnerability concerns

Session Authentication Example (Node.js/Express)

const session = require('express-session');
const MongoStore = require('connect-mongo');

app.use(session({
  secret: process.env.SESSION_SECRET,
  resave: false,
  saveUninitialized: false,
  store: MongoStore.create({
    mongoUrl: process.env.MONGODB_URI
  }),
  cookie: {
    secure: process.env.NODE_ENV === 'production',
    httpOnly: true,
    maxAge: 24 * 60 * 60 * 1000 // 24 hours
  }
}));

// Login route
app.post('/login', async (req, res) => {
  const { username, password } = req.body;
  const user = await User.findOne({ username });
  
  if (user && await bcrypt.compare(password, user.password)) {
    req.session.userId = user._id;
    req.session.username = user.username;
    res.json({ success: true, user: { username: user.username } });
  } else {
    res.status(401).json({ error: 'Invalid credentials' });
  }
});

JWT Authentication

JSON Web Tokens (JWT) are a stateless authentication method where user information is encoded in a token. The token is signed by the server and can be verified without storing session data.

JWT Authentication Flow

sequenceDiagram participant C as Client participant S as Server C->>S: POST /login (username, password) S->>S: Validate credentials S->>S: Generate JWT token S->>C: Return JWT token Note over C: Store token (localStorage/cookie) Note over C,S: Subsequent requests C->>S: Request with JWT in header S->>S: Verify & decode JWT S->>C: Authenticated response

JWT Structure

A JWT consists of three parts separated by dots (.):

  • Header: Token type and signing algorithm
  • Payload: Claims (user data, expiration, etc.)
  • Signature: Ensures token integrity

JWT Authentication Example (Node.js)

const jwt = require('jsonwebtoken');

// Generate JWT token
function generateToken(user) {
  return jwt.sign(
    { 
      userId: user._id, 
      username: user.username,
      email: user.email 
    },
    process.env.JWT_SECRET,
    { expiresIn: '24h' }
  );
}

// Login route
app.post('/login', async (req, res) => {
  const { username, password } = req.body;
  const user = await User.findOne({ username });
  
  if (user && await bcrypt.compare(password, user.password)) {
    const token = generateToken(user);
    res.json({ 
      success: true, 
      token,
      user: { username: user.username, email: user.email }
    });
  } else {
    res.status(401).json({ error: 'Invalid credentials' });
  }
});

// JWT verification middleware
function verifyToken(req, res, next) {
  const token = req.headers.authorization?.split(' ')[1];
  
  if (!token) {
    return res.status(401).json({ error: 'No token provided' });
  }
  
  try {
    const decoded = jwt.verify(token, process.env.JWT_SECRET);
    req.user = decoded;
    next();
  } catch (error) {
    res.status(403).json({ error: 'Invalid token' });
  }
}

OAuth 2.0

OAuth 2.0 is an authorization framework that enables applications to obtain limited access to user accounts on HTTP services. It's commonly used for "Login with Google/Facebook" functionality.

OAuth 2.0 Authorization Code Flow

OAuth 2.0 Implementation Example (Node.js/Passport)

const passport = require('passport');
const GoogleStrategy = require('passport-google-oauth20').Strategy;

passport.use(new GoogleStrategy({
  clientID: process.env.GOOGLE_CLIENT_ID,
  clientSecret: process.env.GOOGLE_CLIENT_SECRET,
  callbackURL: "/auth/google/callback"
}, async (accessToken, refreshToken, profile, done) => {
  try {
    let user = await User.findOne({ googleId: profile.id });
    
    if (user) {
      return done(null, user);
    } else {
      user = await User.create({
        googleId: profile.id,
        name: profile.displayName,
        email: profile.emails[0].value,
        avatar: profile.photos[0].value
      });
      return done(null, user);
    }
  } catch (error) {
    return done(error, null);
  }
}));

// Routes
app.get('/auth/google',
  passport.authenticate('google', { scope: ['profile', 'email'] })
);

app.get('/auth/google/callback',
  passport.authenticate('google', { failureRedirect: '/login' }),
  (req, res) => {
    res.redirect('/dashboard');
  }
);

Security Best Practices

üîí Secure Token Storage

Store tokens securely using httpOnly cookies or secure storage mechanisms. Avoid localStorage for sensitive tokens.

‚è±Ô∏è Token Expiration

Implement short-lived access tokens with longer-lived refresh tokens. Rotate tokens regularly.

üõ°Ô∏è HTTPS Only

Always use HTTPS for authentication endpoints and token transmission. Never send credentials over HTTP.

üîê Strong Secrets

Use cryptographically strong secrets for signing tokens. Store secrets securely and rotate them periodically.

üö´ Input Validation

Validate and sanitize all input. Implement rate limiting to prevent brute force attacks.

üìù Audit Logging

Log authentication events for security monitoring and forensic analysis.

Implementation Examples

Frontend JWT Implementation (React)

import axios from 'axios';

// Token management utilities
const TOKEN_KEY = 'auth_token';

export const authService = {
  // Store token
  setToken(token) {
    localStorage.setItem(TOKEN_KEY, token);
    axios.defaults.headers.common['Authorization'] = `Bearer ${token}`;
  },

  // Get stored token
  getToken() {
    return localStorage.getItem(TOKEN_KEY);
  },

  // Remove token
  removeToken() {
    localStorage.removeItem(TOKEN_KEY);
    delete axios.defaults.headers.common['Authorization'];
  },

  // Login function
  async login(credentials) {
    try {
      const response = await axios.post('/api/login', credentials);
      const { token, user } = response.data;
      
      this.setToken(token);
      return { success: true, user };
    } catch (error) {
      return { success: false, error: error.response.data.message };
    }
  },

  // Logout function
  logout() {
    this.removeToken();
    window.location.href = '/login';
  },

  // Check if user is authenticated
  isAuthenticated() {
    const token = this.getToken();
    if (!token) return false;
    
    try {
      const payload = JSON.parse(atob(token.split('.')[1]));
      return payload.exp > Date.now() / 1000;
    } catch {
      return false;
    }
  }
};

// Axios interceptor for token refresh
axios.interceptors.response.use(
  (response) => response,
  async (error) => {
    if (error.response?.status === 401) {
      authService.logout();
    }
    return Promise.reject(error);
  }
);
